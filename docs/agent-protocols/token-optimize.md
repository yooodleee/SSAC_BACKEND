# Token Optimization Protocol (BE)

## 트리거 조건
- 스프린트 종료 시점 (수동)
- 사용자가 "느려", "토큰", "최적화" 언급 시 (자동)
- harness-audit.md 실행 결과 이상 감지 시 (자동)

## 실행 순서
STEP 1. 컨텍스트 최소화
STEP 2. 미사용 파일 감사
STEP 3. 프로토콜 중복 제거
STEP 4. 최적화 결과 기록

---

## STEP 1. 컨텍스트 최소화

### 작업 시작 전 필수 점검
□ 현재 작업과 직접 관련된 파일만 로드한다
□ 아래 파일은 명시적 요청 없이 로드하지 않는다

로드 금지 파일 목록 (작업 무관 시):
- src/main/resources/application-prod.yml
- docs/debug-log.md (진단 작업 외)
- build/reports/jacoco/** (커버리지 확인 작업 외)
- .gradle/** (빌드 설정 변경 작업 외)
- src/test/** (테스트 작업 외)

### 파일 로드 우선순위
1순위 : 현재 작업 SC와 직접 연관된 파일
2순위 : 해당 파일이 import하는 파일
3순위 : Contract 파일 (API 연동 작업 시)
4순위 : 프로토콜 파일 (트리거 조건 충족 시)

→ 4순위 이하 파일은 로드하지 않는다

## STEP 2. 미사용 파일 감사

아래 기준으로 미사용 파일을 식별한다:

### 즉시 삭제 대상
□ 최근 30일 이상 수정되지 않은 임시 파일
  → 패턴: *.tmp, *_backup.*, *_old.*
□ 주석 처리된 코드 블록이 50줄 이상인 파일
□ 아무 곳에서도 import되지 않는 클래스 파일
  → 검증: grep -r "import.*{파일명}" src/
□ 빈 테스트 파일 (테스트 메서드가 0개인 파일)

### 검토 후 삭제 대상
□ TODO 주석이 90일 이상 방치된 파일
□ @Deprecated 어노테이션이 붙은 클래스
□ 사용되지 않는 DTO 클래스

### 삭제 전 확인 절차
1. git log --oneline -5 {파일명} 으로 최근 커밋 확인
2. grep -r "{파일명}" . 으로 참조 여부 확인
3. 참조 없음 확인 후 삭제
4. 삭제 후 빌드 성공 확인 (./gradlew build)

출력 형식:
[미사용 파일 감사 결과]
즉시 삭제 : 3개
- src/main/java/.../OldNewsDto.java (미참조 60일)
- src/main/java/.../TempService.java (미참조 45일)
검토 필요 : 2개
- src/main/java/.../DeprecatedTokenUtil.java (@Deprecated)

## STEP 3. 프로토콜 중복 제거

아래 기준으로 프로토콜 파일 중복을 식별한다:

□ 동일한 체크리스트 항목이 2개 이상의 프로토콜에 존재하는가
  → 공통 항목을 CLAUDE.md 전역 규칙으로 이동
  → 개별 프로토콜에서 해당 항목 삭제

□ 프로토콜 파일이 500줄을 초과하는가
  → 핵심 규칙만 남기고 예시 코드를 별도 파일로 분리
  → 예시: docs/examples/testing-examples.md

□ 트리거 조건이 중복되는 프로토콜이 있는가
  → 중복 트리거를 가진 프로토콜을 병합 검토

프로토콜 파일 적정 크기 기준:
- 핵심 규칙 파일 : 200줄 이하
- 예시 포함 파일 : 500줄 이하
- 500줄 초과 시  : 분리 필수

출력 형식:
[프로토콜 중복 감사 결과]
중복 항목 : 5개
- validateErrorContract 체크 (be-log-diagnose.md / harness-audit.md)
분리 필요 : 1개
- token-optimize.md (현재 620줄 → 분리 권장)

## STEP 4. 최적화 결과 기록

실행 완료 후 결과를 docs/optimization-log.md에 아래 형식으로 기록한다:

```
## [날짜] 토큰 최적화 기록

### 컨텍스트 최소화
- 제거된 불필요 로드 파일 : N개

### 미사용 파일 감사
- 즉시 삭제 : N개
- 검토 후 삭제 : N개

### 프로토콜 중복 제거
- 제거된 중복 항목 : N개
- 분리된 파일 : N개

→ 예상 토큰 절감율 : N%
```
