# ADR 002: 핵심 원칙 #1 — Agent 중심 아키텍처

**상태**: 채택됨  
**날짜**: 2026-04-18  
**담당자**: 팀 전체

---

## 컨텍스트

이 프로젝트는 인간 개발자와 AI 에이전트(Claude Code 등)가 함께 협업하는 구조로 설계되었다.
하네스 시스템의 모든 설계 결정은 이 협업 방식을 전제로 내려져야 한다.

---

## 결정 사항

### 핵심 원칙 #1: Agent 중심 아키텍처 (Agent-Centric Architecture)

**인간은 의도(intent)만 정의하고, 실제 구현은 AI 에이전트가 수행한다.**

---

## 원칙의 의미

### 인간의 역할 — 의도(Intent) 정의

인간 개발자가 담당하는 것:

- **무엇을** 만들어야 하는지 (기능 요구사항, 비즈니스 로직)
- **왜** 만들어야 하는지 (목적, 우선순위, 제약 조건)
- **어떤 기준**으로 완료를 판단하는지 (완료 조건, 검수 기준)
- 에이전트가 내린 결정에 대한 **최종 승인 또는 거부**

인간 개발자가 담당하지 않는 것:

- 클래스/메서드 직접 작성
- 보일러플레이트 코드 타이핑
- 패키지 구조 세부 배치
- 의존성 import 순서 정리

### 에이전트의 역할 — 구현(Implementation) 수행

에이전트가 담당하는 것:

- 인간이 정의한 의도를 코드로 변환
- 컨벤션(`docs/conventions.md`) 및 아키텍처(`docs/architecture.md`) 규칙 준수
- 테스트 코드 작성 및 빌드 검증
- ADR 작성 및 문서 유지

---

## 설계 원칙에 대한 영향

이 원칙에 따라 하네스 시스템의 모든 설계는 다음 기준으로 평가된다:

| 기준 | 올바른 방향 | 잘못된 방향 |
|------|------------|------------|
| API 설계 | 에이전트가 의도를 읽고 엔드포인트를 생성한다 | 인간이 직접 URL 경로를 타이핑한다 |
| 규칙 강제 | 기계적 검증(ArchUnit, Checkstyle)으로 자동화한다 | 코드 리뷰 시 눈으로 확인한다 |
| 문서화 | 에이전트가 변경과 동시에 ADR/주석을 작성한다 | 개발 후 별도 문서화 작업을 한다 |
| 오류 수정 | 에이전트가 빌드 실패를 감지하고 자동 수정한다 | 인간이 로그를 읽고 직접 수정한다 |

---

## 금지 행동 (이 원칙 위반)

- 에이전트 없이 수동으로 코드를 작성하는 것을 전제로 규칙을 설계하는 것
- "개발자가 직접 확인해야 한다"는 전제 하에 자동화를 생략하는 것
- 에이전트가 이해할 수 없는 암묵적 합의에 의존하는 것 (문서화 필수)
- 의도 정의 없이 에이전트에게 구현을 맡기는 것 (방향 없는 위임)

---

## 채택 이유

- 팀 규모(3인)와 개발 속도를 고려할 때 에이전트 위임이 가장 효율적이다
- 수동 코딩 전제 설계는 에이전트와 충돌하거나 이점을 낭비한다
- 의도와 구현의 역할 분리는 책임 소재를 명확히 한다

---

## 재검토 조건

- AI 에이전트 도구의 능력 한계로 자동 구현이 불가능한 영역이 발생할 때
- 팀이 에이전트 협업 방식에서 전통적 개발 방식으로 전환을 결정할 때
